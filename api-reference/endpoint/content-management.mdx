## Overview

The Content Management endpoints allow you to retrieve generation history, manage saved images, and organize your content library.

## Get Generation History

### Endpoint

```http
GET /v1/content/history
```

### Query Parameters

<ParamField path="limit" type="integer">
  Number of results to return (1-100). Default: 20
</ParamField>

<ParamField path="offset" type="integer">
  Number of results to skip. Default: 0
</ParamField>

<ParamField path="tool" type="string">
  Filter by AI tool used. Options: `text-to-image`, `image-to-image`, `sketch-to-image`, `inpainting`, `upscale`
</ParamField>

<ParamField path="date_from" type="string">
  Start date for filtering (ISO 8601 format)
</ParamField>

<ParamField path="date_to" type="string">
  End date for filtering (ISO 8601 format)
</ParamField>

### Response

<ResponseField name="data.history" type="array">
  Array of generation history items
</ResponseField>

<ResponseField name="data.history[].id" type="string">
  Generation ID
</ResponseField>

<ResponseField name="data.history[].tool" type="string">
  AI tool used
</ResponseField>

<ResponseField name="data.history[].prompt" type="string">
  Prompt used for generation
</ResponseField>

<ResponseField name="data.history[].image_url" type="string">
  URL to generated image
</ResponseField>

<ResponseField name="data.history[].created_at" type="string">
  Generation timestamp
</ResponseField>

<ResponseField name="data.history[].status" type="string">
  Generation status (completed, failed, processing)
</ResponseField>

<ResponseField name="data.pagination" type="object">
  Pagination information
</ResponseField>

<ResponseField name="data.pagination.total" type="integer">
  Total number of results
</ResponseField>

<ResponseField name="data.pagination.has_more" type="boolean">
  Whether there are more results available
</ResponseField>

## Get Saved Images

### Endpoint

```http
GET /v1/content/saved
```

### Query Parameters

<ParamField path="limit" type="integer">
  Number of results to return (1-100). Default: 20
</ParamField>

<ParamField path="offset" type="integer">
  Number of results to skip. Default: 0
</ParamField>

<ParamField path="collection" type="string">
  Filter by collection name
</ParamField>

### Response

<ResponseField name="data.saved_images" type="array">
  Array of saved images
</ResponseField>

<ResponseField name="data.saved_images[].id" type="string">
  Saved image ID
</ResponseField>

<ResponseField name="data.saved_images[].generation_id" type="string">
  Original generation ID
</ResponseField>

<ResponseField name="data.saved_images[].image_url" type="string">
  URL to saved image
</ResponseField>

<ResponseField name="data.saved_images[].collection" type="string">
  Collection name
</ParamField>

<ResponseField name="data.saved_images[].tags" type="array">
  Tags associated with the image
</ResponseField>

<ResponseField name="data.saved_images[].notes" type="string">
  User notes for the image
</ResponseField>

<ResponseField name="data.saved_images[].saved_at" type="string">
  Save timestamp
</ResponseField>

## Save Image

### Endpoint

```http
POST /v1/content/save
```

### Request Body

<ParamField path="generation_id" type="string" required>
  ID of the generation to save
</ParamField>

<ParamField path="collection" type="string">
  Collection name to save to
</ParamField>

<ParamField path="tags" type="array">
  Array of tags to associate with the image
</ParamField>

<ParamField path="notes" type="string">
  User notes for the image
</ParamField>

### Example Request

```bash
curl -X POST https://api.omniphotostudio.com/v1/content/save \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "generation_id": "img_123456789",
    "collection": "My Favorites",
    "tags": ["landscape", "sunset"],
    "notes": "Beautiful mountain landscape"
  }'
```

## Delete Image

### Endpoint

```http
DELETE /v1/content/delete
```

### Request Body

<ParamField path="image_id" type="string" required>
  ID of the image to delete
</ParamField>

<ParamField path="type" type="string" required>
  Type of image to delete. Options: `generation`, `saved`
</ParamField>

### Example Request

```bash
curl -X DELETE https://api.omniphotostudio.com/v1/content/delete \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "image_id": "img_123456789",
    "type": "generation"
  }'
```

## Error Responses

### Image Not Found

```json
{
  "success": false,
  "error": {
    "code": "image_not_found",
    "message": "Image not found",
    "details": "The specified image ID does not exist or is not accessible"
  }
}
```

### Insufficient Permissions

```json
{
  "success": false,
  "error": {
    "code": "insufficient_permissions",
    "message": "Insufficient permissions",
    "details": "You do not have permission to perform this action"
  }
}
```

## Rate Limits

Content management endpoints are subject to rate limiting based on your subscription plan. See the [Rate Limits](/api-reference/introduction#rate-limits) section for details.

## Best Practices

- Use pagination for large result sets
- Filter results to improve performance
- Organize content with meaningful collections and tags
- Add descriptive notes for better content organization
- Regularly clean up unused content to manage storage
